Distanta Levenshtein

Enunţ

Fie un şir oarecare de caractere. Asupra lui definim următoarele 3 operaţii:
  • inserare: pe o poziţie oarecare se adaugă un caracter oarecare
  • ştergere: se şterge caracterul de pe o poziţie oarecare din şir
  • înlocuire: se înlocuieşte caracterul de pe o poziţie oarecare din şir cu orice alt caracter.
Se dau două şiruri S1 şi S2 conţinând doar litere mici ale alfabetului englez şi având N şi, respectiv, M caractere. Distanţa de editare a celor două şiruri este numărul minim de operaţii definite mai sus ce trebuie aplicat asupra primului şir pentru a-l transforma în cel de-al doilea şir. Acelaşi tip de operaţie poate fi aplicat de mai multe ori.

Cerinţă

Aflaţi distanţa de editare a celor doua şiruri.

Date de intrare

Pe prima linie a fişierului de intrare in.txt se află doua numere întregi: N si M. Pe a doua linie a fişierului se află N litere mici din alfabetul englez reprezentând şirul S1. Pe a treia linie a fişierului se află M litere mici din alfabetul englez ce reprezintă şirul S2.

Date de ieşire

În fişierul de ieşire out.txt se va afla un singur număr întreg reprezentând distanţa de editare a celor doua şiruri.

in.txt out.txt Explicatie

Descrierea soluţiei

Această problemă se poate rezolva folosind formula de recurenţă Levenshtein. Avem de calculat numărul minim de paşi pe care trebuie să îl parcurgem pentru a ajunge de la un şir de caractere la altul. Pentru aceasta, definim cele trei operaţii menţionate mai sus: ştergerea unui caracter, adăugarea unui caracter şi interschimbarea unui caracter cu altul. După fiecare parcurgere, rezultatul va fi memorat într-un tablou bidimensional, o matrice, numită formal dinamică. Rezultatul final, ceea ce se va afişa în fişierul out.txt, este elementul care se găseşte pe linia m+1 şi coloana n+1 al dinamicei.
Spre exemplu, încercăm transformarea cuvântului sac în cuvântul mare. Pentru aceasta, se vor realiza următoarele operaţii:
  • sac => mac (schimbarea literei s cu litera m)
  • mac => mar (schimbarea literei c cu litera r)
  • mar => mare (adăugarea literei e)
Putem defini cele trei operaţii elementare prin următoarea relaţie de recurenţă:
lev(a,b)(i, j) =
  • max(i, j), daca min(i, j) = 0
  • min(lev(a,b)(i-1, j)+1, lev(a,b)(i, j-1)+1, lev(a,b)(i-1, j-1)+[ai≠bj]), altfel
Prima ramură a recurenţei reprezintă defapt iniţializarea matricei.
Cele trei operaţii se definesc astfel:
  • lev(a,b)(i-1, j)+1: ştergerea unui element
  • lev(a,b)(i, j-1)+1: adăugarea unui element
  • lev(a,b)(i-1, j-1)+[ai≠bj]: interschimbarea elementului ai cu bj, dacă acestea două sunt diferite. [ai≠bj] are valoarea 1 dacă ai≠bj , şi 0 dacă ai= bj.

Observăm că la fiecare pas avem nevoie doar de două linii, linia curentă si cea dinaintea ei. Pentru a optimiza memoria utilizată, putem folosi o matrice cu doar două linii, construind alternativ pe fiecare din cele două linii. Pentru aceasta, vom folosi o variabilă booleană, care va avea alternativ valoarea true, respectiv false.

Complexitatea algoritmului

Considerând optimizarea adusă anterior bazată pe observaţia că se folosesc doar două rânduri din matrice pentru luarea unei decizii, tabloul alocat va avea două linii şi n coloane, n fiind lungimea primului şir. Pe lângă tabloul-dinamică, mai avem nevoie de două şiruri de caractere pentru memorarea şirurilor date în problemă. Luând ca operaţie elementară atribuirea, complexitatea algoritmului este O(n*m), deci spunem că este un algoritm pătratic.
